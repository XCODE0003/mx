# Реферальная система

## Описание

Реферальная система позволяет отслеживать переходы пользователей по партнерским ссылкам, регистрации и покупки через реферальные источники.

## Возможности

1. **Создание реферальных ссылок** - в админке можно создавать уникальные реферальные ссылки для разных партнеров
2. **Отслеживание переходов** - система автоматически фиксирует каждый переход по реферальной ссылке
3. **Отслеживание регистраций** - при регистрации пользователя сохраняется информация о том, по какой реферальной ссылке он пришел
4. **Отслеживание покупок** - при создании заказа сохраняется информация о реферальной ссылке
5. **Статистика** - детальная статистика по каждой реферальной ссылке:
   - Количество кликов
   - Количество регистраций
   - Количество покупок
   - Общая выручка

## Структура базы данных

### Таблица `referral_links`
- `id` - ID ссылки
- `code` - уникальный код ссылки
- `name` - название партнера/источника
- `description` - описание
- `clicks` - счетчик кликов
- `is_active` - активна ли ссылка
- `created_at`, `updated_at`

### Таблица `referral_visits`
- `id` - ID визита
- `referral_link_id` - ID реферальной ссылки
- `user_id` - ID пользователя (если авторизован)
- `ip_address` - IP адрес посетителя
- `user_agent` - User Agent браузера
- `visited_at` - время визита

### Добавлено в `users`
- `referral_link_id` - ID реферальной ссылки, по которой зарегистрировался

### Добавлено в `orders`
- `referral_link_id` - ID реферальной ссылки, по которой совершена покупка

## Использование

### Создание реферальной ссылки

1. Перейдите в админку: `/admin/referrals`
2. Заполните форму создания ссылки:
   - **Название партнера** - обязательное поле (например, "Партнер Иванов")
   - **Код** - опционально, если не указать, будет сгенерирован автоматически
   - **Описание** - опционально
3. Нажмите "Создать ссылку"

### Формат ссылки

Реферальная ссылка имеет формат: `https://kim365.ru/?ref=КОД`

Например: `https://kim365.ru/?ref=partner123`

### Просмотр статистики

1. В списке реферальных ссылок (`/admin/referrals`) отображается общая статистика по каждой ссылке
2. Нажмите "Подробнее" для просмотра детальной статистики:
   - Полная история переходов с IP адресами
   - Список зарегистрировавшихся пользователей
   - Список совершенных покупок с суммами

### Просмотр пользователей

В разделе `/admin/users` отображается список всех пользователей с информацией о том, по какой реферальной ссылке они пришли.

## Как работает система

1. **Переход по ссылке:**
   - Пользователь переходит по ссылке `https://kim365.ru/?ref=CODE`
   - Middleware `TrackReferral` перехватывает параметр `ref`
   - Код сохраняется в сессии пользователя
   - Увеличивается счетчик кликов
   - Создается запись в `referral_visits`

2. **Регистрация:**
   - При регистрации проверяется наличие кода в сессии
   - Если код есть, `referral_link_id` сохраняется в таблице `users`

3. **Покупка:**
   - При создании заказа проверяется наличие кода в сессии
   - Если код есть, `referral_link_id` сохраняется в таблице `orders`

## API Endpoints (админка)

- `GET /admin/referrals` - список реферальных ссылок
- `POST /admin/referrals` - создание новой ссылки
- `GET /admin/referrals/{id}` - детальная статистика по ссылке
- `PUT /admin/referrals/{id}` - обновление ссылки (активация/деактивация)
- `DELETE /admin/referrals/{id}` - удаление ссылки
- `GET /admin/users` - список пользователей с реферальными данными

## Доступ

Доступ к админке имеют только пользователи с `is_admin = 1`.
