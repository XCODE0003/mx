# 📧 Настройка отправки писем на КИМ365

## ✅ Что настроено

### 1. SMTP-сервер Mail.ru
Настройки в `.env`:
```env
MAIL_MAILER=smtp
MAIL_HOST=smtp.mail.ru
MAIL_PORT=465
MAIL_USERNAME=support@kim365.ru
MAIL_PASSWORD=ejEAUAIZsohIBxesHQae
MAIL_ENCRYPTION=ssl
MAIL_FROM_ADDRESS="support@kim365.ru"
MAIL_FROM_NAME="КИМ365"
```

### 2. Восстановление пароля

**Письмо:** `ResetPasswordNotification`
- Отправляется автоматически при запросе сброса пароля
- Содержит ссылку, действительную 60 минут
- Стилизовано в фирменных цветах КИМ365

**Маршруты:**
- `GET /forgot-password` — форма запроса сброса
- `POST /forgot-password` — отправка письма
- `GET /reset-password/{token}` — форма ввода нового пароля
- `POST /reset-password` — сохранение нового пароля

### 3. Подтверждение email

**Письмо:** `VerifyEmailNotification`
- Отправляется автоматически после регистрации
- Содержит подписанную ссылку для верификации
- Стилизовано в фирменных цветах КИМ365

**Маршруты:**
- `GET /verify-email/{id}/{hash}` — подтверждение email (защищено подписью)
- `POST /email/verification-notification` — повторная отправка письма (лимит 6 запросов в минуту)

### 4. Model User
Обновлена модель User:
- Добавлен `MustVerifyEmail` interface
- Переопределены методы отправки уведомлений
- При регистрации автоматически отправляется письмо для подтверждения

## 🔒 Безопасность

### Защита от спама
- Throttling на маршруты верификации (6 запросов в минуту)
- Подписанные URL для подтверждения email
- Временные токены для сброса пароля (60 минут)

### Рекомендации
1. **Регулярно меняйте пароль приложения** Mail.ru
2. **Мониторьте отправку писем** через логи Laravel
3. **Настройте лимиты** на количество писем в день (если требуется)

## 📝 Использование

### При регистрации пользователя
```php
$user = User::create([...]);
event(new Registered($user)); // Автоматически отправляет письмо
```

### Восстановление пароля
```php
// В контроллере (уже настроено)

```

### Повторная отправка письма верификации

## 🧪 Тестирование

### Проверка отправки писем
1. Зарегистрируйте нового пользователя
2. Проверьте `storage/logs/laravel.log` на наличие ошибок
3. Проверьте почту `support@kim365.ru` на наличие отправленных писем

### Проверка восстановления пароля
1. Перейдите на `/forgot-password`
2. Введите email существующего пользователя
3. Проверьте почту на получение письма
4. Перейдите по ссылке и задайте новый пароль

### Проверка подтверждения email
1. Зарегистрируйте новый аккаунт
2. Проверьте почту на получение письма
3. Перейдите по ссылке для подтверждения
4. Убедитесь, что `email_verified_at` заполнен в базе данных

## ⚙️ Настройки Mail.ru

### Данные серверов
- **SMTP-сервер:** smtp.mail.ru
- **Порт:** 465 (SSL/TLS)
- **Протокол:** SSL

### Создание пароля приложения
1. Откройте Настройки → Безопасность
2. Перейдите в "Пароли для внешних приложений"
3. Создайте пароль с названием "КИМ365 Laravel"
4. Выберите "Полный доступ к Почте"
5. Скопируйте пароль в `.env`

### Важно
- Не используйте обычный пароль от почты — только пароль приложения
- Убедитесь, что к почте привязан телефон
- После удаления пароля приложения письма перестанут отправляться

## 🐛 Troubleshooting

### Письма не отправляются
1. Проверьте логи: `tail -f storage/logs/laravel.log`
2. Убедитесь, что пароль приложения актуален
3. Проверьте настройки `.env`
4. Очистите кэш: `php artisan config:clear`

### Письма попадают в спам
1. Настройте SPF записи для домена kim365.ru
2. Настройте DKIM подпись
3. Убедитесь, что `MAIL_FROM_ADDRESS` = `MAIL_USERNAME`

### Ошибка "Connection refused"
- Проверьте, что порт 465 не заблокирован файрволлом
- Попробуйте использовать порт 587 с `MAIL_ENCRYPTION=tls`
